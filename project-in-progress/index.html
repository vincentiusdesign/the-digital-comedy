      The Shore

      After the descent, a shore.

      No gate, no inscription. The mountain rises from the water in early light — gray before dawn, the kind of quiet that follows duration. You have been somewhere enclosed for a long time.

      The water here is not the Acheron. It does not carry you toward punishment. It simply arrives at land.

      Virgil wipes the grime of Hell from Dante’s face with the morning dew. A reed he picks from the sand grows back instantly. This realm operates by different rules. What is taken is returned. What is erased is not lost.

      Somewhere between the shore and the first terrace, there is an Ante-Purgatory — the late repentant, those who came to their reckoning at the last possible moment. They wait here without punishment, simply without urgency. There is time now.

      The mountain has seven terraces. Each one is a problem that was actually worked on.

      Purgatory

      Terrace 1 — Pride

      The Layout Problem

      Early HTML, ~1991–1997

      THE CONSTRAINT
      There is no CSS.

      Let that settle for a moment. There is no document object model as you understand it. There is no box model. There is no semantic distinction between presentation and structure. There are tags — heading tags, paragraph tags, a horizontal rule — and there is a FONT tag that will do approximately what you need if you nest it correctly, and there are TABLE tags, and if you are clever enough with TABLE tags you can make something that looks, in most browsers, like a layout.

      Most browsers. The phrase will become important. In 1994 there are perhaps four browsers that matter. They disagree about almost everything.

      This is the first problem: the page has no inherent spatial logic. Images and text flow together, one after another, like words in a document. But this is not a document. This is a page, and pages have columns and grids and alignment, and none of those things exist yet in any formal sense.

      The solution, arrived at by necessity and shared between designers who had no common vocabulary for what they were doing, was the table. Not a table for tabular data — a table as structural skeleton. Nested inside itself at two, three, four levels of depth. Cells stretched with single-pixel transparent GIFs — spacer GIFs — because a 1×1 pixel image set to a specific width holds space in a way that an empty cell will not.

      This is not a hack, exactly. It is engineering under constraint. The constraint is complete.

      What was accomplished inside it was remarkable.
      [ THE CODE BELOW PRODUCED THIS LAYOUT ]

      <table border="0" cellpadding="0" cellspacing="0" width="100%">
        <tr>
          <td width="40"><img src="spacer.gif" width="40" height="1"></td>
          <td>
            <font face="Times New Roman" size="6">Heading</font>

      Terrace 2 — Envy

      The Control Problem

      Flash, ~1996–2005

      THE CONSTRAINT

      In 1999, there was a way to make the web behave like anything you could imagine.

      You could define a timeline. You could place objects at specific coordinates — not approximately, not depending on the browser’s interpretation of your intent, but precisely. You could animate along that timeline, with easing curves, with sound, with video. You could use any typeface you owned. You could make the browser do things the browser had never been designed to do.

      This was Flash. It was beautiful. It was also a walled garden.

      The web outside Flash was open: any browser could read it, any search engine could index it, any screen reader could parse it, any device with a standards-compliant renderer could display it. Flash was none of these things. Flash was a plugin — a piece of software your browser trusted to do whatever it was asked inside a rectangle on the page. Screen readers could not enter that rectangle. Search engines could not enter that rectangle. Mobile devices, in the years when they mattered, could not run it at all.

      Some designers knew this and accepted the trade. Some designers knew this and thought the trade was worth it. Some designers did not know this, because the tools did not make it visible.

      You are viewing this in a browser that does not support Flash.

      You are viewing this in a browser that ended Flash.

      It was right to end it.

      The work was still beautiful.

      Standards-compliant browser

      The Box Model

      Width: 200px

      Padding: 10px

      Total: 220px

      The declared width refers to the content area only. Padding and border are added externally. This is the W3C specification.

      Internet Explorer 5.5

      The Box Model

      Width: 200px

      Padding: 10px

      Total: 200px

      The declared width includes padding and border. The same box renders 20 pixels narrower. This is not a small difference. This is the layout breaking.

      KNOWN WORKAROUNDS
      The Box Model Hack

      Tantek Çelik, 2000: exploit a CSS parsing bug in IE to serve different values to IE and standards-compliant browsers simultaneously. The hack works. It requires hiding valid CSS from a browser using another browser’s bug. This is the industry standard solution.

      div.box {
  width: 220px; /* IE5.5 sees this */
  voice-family: "\"}\"";
  voice-family: inherit;
  width: 200px; /* standards-compliant browsers see this */
}
html>body div.box {
  width: 200px;
}
      Float Clearing

      When floated elements overflow their container, the container collapses to zero height. The fix involves inserting an element after the floats and applying clear: both. Alternatively: apply overflow: hidden to the container, which clears floats as a side effect of a property that has nothing to do with floats. This is also the industry standard solution.

      Conditional Comments

      Microsoft Internet Explorer supports a comment syntax that only IE can read. Use this to serve IE-specific stylesheets that undo what the standard stylesheet does. This is the third industry standard solution.

      <!--[if IE]>
  <link rel="stylesheet" href="ie-only.css">
<![endif]-->

      Terrace 5 — Avarice

      The Scale Problem

      Framework Age, ~2010–2016

      Introducing Bootstrap

      A sleek, intuitive, and powerful front-end framework for faster and easier web development.

      Bootstrap was released by Twitter in 2011. Within two years it was the most starred project on GitHub. The 12-column grid it established became the implicit default — the assumed structure underlying almost every site built during this period. Not because Twitter mandated it. Because it was there, it worked, and it was already installed.

      What it gave you:

      A responsive grid before responsive was easy. Consistent button styles across browsers. A dropdown component that worked the first time. A navbar that collapsed on mobile before you had to think about mobile. Accessibility defaults baked into components you were going to build anyway. Documentation. A community. Answers on Stack Overflow.

      What it took:

      The moment when a site felt like someone made it. The particular decision, the unexplainable choice, the element that doesn’t belong in a grid but belongs on this page. Design by a person, for a reason.

      The framework did not mandate these trades. The framework was a choice. The choice calcified into an assumption — and the assumption, once invisible, began to make choices of its own.

      Terrace 6 — Gluttony

      The Device Problem

      Responsive Era, ~2014–2020

      This text was designed for every screen simultaneously.

      Fluid grids — elements that resize proportionally rather than snapping to fixed widths. Breakpoints — the specific pixel widths at which a layout reorganizes itself. Media queries — the CSS rule that makes breakpoints possible, asking the browser: how wide are you right now? Images that resize without distortion. Type that scales. Navigation that collapses.

      This was a genuine problem with a genuine solution. The proliferation of screen sizes in the early 2010s made fixed-width design increasingly incoherent. A site built for 960 pixels wide looks wrong on a phone and cramped on a widescreen monitor. Responsive design was the answer.

      But designing for every screen simultaneously is also designing for no screen in particular.

      A fixed-width site is wrong on half of all devices and exactly right on a specific rectangle of glass that someone is holding. A responsive site is approximately right everywhere, by design. The specific rectangle disappears. The place disappears.

      You are reading this wherever you are.

      On the other side: the Earthly Paradise. The garden at the summit, before the spheres begin.

      Heaven

      Sphere 1 — The Moon

      The Democratic Web

      Anyone. This was the word that mattered.

      Not anyone with a printing press, anyone with a broadcast license, anyone with a publisher, anyone with an editor, anyone with a masthead and a distribution network.

      Anyone.

      In 1995 a person could build a page. Not a page in a newspaper. Not a page in a directory. A page — their page — on a medium that had no center and therefore no periphery, no masthead and therefore no gate, no gatekeepers and therefore no one to keep you out.

      The page was often garish. The page was often earnest. The page was often about things that no editor would have approved — the particular fandom, the personal grief, the collection of links that only made sense to one person’s mind.

      The garish was the point. The earnest was the point.

      Anyone could make a page and anyone could see it, and those two facts together had never been true before — for anything, ever.

      Sphere 2 — Mercury

      The Informational Web

      Imagine a world in which every single person on the planet is given free access to the sum of all human knowledge.

      That’s what we’re doing.

      Not a version of all human knowledge curated by an institution. Not a version vetted by a single editorial tradition. A version written by everyone who knew something, corrected by everyone who knew better, maintained by everyone who believed it should exist.

      The encyclopedia dream was always this: not one book, but the principle of the book applied to everything. Cross-referenced. Living. Correctable. Not finished, because knowledge is not finished.

      This sphere does not answer what it would actually mean if this worked. If every person on earth had genuine access to everything any other person had thought to write down.

      That question is still open.

      Sphere 3 — Venus

      The Social Web

      You knew them by their handle first.

      Before their name, before their face, before any of the information that the later web would require you to provide as proof of personhood — before all of that, there was a handle. A chosen name for a chosen self. And around that self: a signature, a set of fandoms, a list of links to other people’s pages, a particular way of writing.

      The community had in-jokes you learned by being there long enough. It had a texture — the specific texture of people who had found each other across geography because they were looking for the same thing, and the web was briefly the only place that thing existed.

      This was before scale became the point.

      This sphere is not an elegy. It is a record of what intimacy at scale felt like, before scale was all there was.

      Sphere 4 — The Sun

      The Open Web

      This is for everyone.

      Not for everyone who registers. Not for everyone who agrees to the terms. Not for everyone who uses the approved client or the approved device. For everyone. As a fact of its architecture.

      The web I imagined was a medium that no one owned — built on protocols that any implementation could speak, readable by any device that understood them, writable by anyone who learned the language. Not free as in no cost. Free as in no permission required. Free as in the road, which any vehicle can use, which belongs to no vehicle.

      What I did not anticipate was that you could build a private tollway on top of the road and call it a destination.

      The road is still there.

      The road is still free.

      — Tim Berners-Lee, 1989

      Sphere 5 — Mars

      The Creative Web

      JODI made a website in 1994 that looked broken.

      It was not broken. It was a work of art about the experience of the web — the error messages, the source code visible when something failed, the browser as a medium with its own language and its own failures. They made the machinery visible and called it a canvas.

      Olia Lialina made a love story in hypertext in 1996 that could only be told in hypertext — where clicking was grief, and navigation was longing, and the fact that you could leave at any moment was part of what it meant that you stayed.

      Young-Hae Chang Heavy Industries made text flash at you in time to music at a speed that forced you to read their way or not at all.

      These were not websites about art. They were art about websites — made for a medium that had never existed before and therefore had never been used before. The properties of the medium were the material: the link, the error, the load time, the cursor, the scroll.

      This sphere is not a representation of net art.

      This sphere is net art.

      Sphere 6 — Jupiter

      The Accessible Web

      The web is for everyone.

      Not as a slogan. As a specification. The markup language was designed so that a screen reader could parse it, so that a user who cannot use a mouse could navigate it with a keyboard, so that a person on a slow connection could receive it without the decorative parts, so that a person with a visual impairment could receive a text description of what others see.

      The tools existed from the beginning. The will to use them was inconsistent.

      This sphere practices what it describes. The experience of a screen reader user here is equal to the experience of a sighted user — not a degraded alternative, not an afterthought, not an aria-label added at the end. Equal.

      The vision was not achieved. That is not an argument against the vision. It is an argument for it: the idealism was right, and the falling short was not inevitable. Someone decided not to. Someone can decide otherwise.

      The web this sphere describes still does not fully exist. That is why it belongs in Heaven.

      Sphere 7 — Saturn

      The Indie Web

      Own your domain.

      Not the platform’s domain. Not the subdomain they assigned you when you signed up. Your domain — the address that is yours because you chose it and paid for it and no company’s decision to pivot or shut down or get acquired can take it from you.

      Publish to your own site. Syndicate everywhere else. Let your words and your photos and your thoughts live on infrastructure you control, in formats you own, that do not depend on any company’s continued existence or good intentions.

      This is not utopian. This is not a vision of the web that was — it never quite was. It is a vision of the web that can still be, practiced now, by people who noticed what was being lost and decided not to lose it.

      It is smaller than the web most people use. It is quieter. It does not have an algorithm.

      The people on it know your name because they chose to.

      Sphere 8 — The Fixed Stars

      The Web as Commons

      The Internet Archive has been preserving the web since 1996.

      Not because anyone asked them to. Not because anyone paid them to preserve it, specifically. Because someone understood that digital things disappear — not slowly, the way stone erodes, but suddenly, without announcement, when a server is shut down or a company folds or a format becomes unreadable.

      The Archive is a commons in the technical sense: a shared resource maintained collectively, whose value belongs to everyone who contributed to it and everyone who will use it. Its value is not in any single item it holds. Its value is in the principle it embodies — that the record of what we made deserves to exist past the moment of its making.

      Open source code. Creative Commons licenses. The Wayback Machine. These are not products. They are agreements: an agreement that some things should belong to everyone who needs them, that the cost of enclosing them is greater than the cost of maintaining them, that the web’s value was never in any platform but in the whole.

      The whole still exists. It requires maintenance.

      Sphere 9 — The Primum Mobile

      The Semantic Web

      I have a dream for the Web.

      It is a web in which computers become capable of analyzing all the data on the Web — the content, the links, the transactions between people and computers. A Semantic Web that makes this possible has yet to emerge, but when it does, the day-to-day mechanisms of trade, bureaucracy, and our daily lives will be handled by machines talking to machines.

      The vision requires that data be given meaning — not just structure, but meaning. That a machine encountering a word could know what it referred to in context. That the relationships between facts were as machine-readable as the facts themselves. That the web could reason about what it contained.

      It was technically possible. It remained technically difficult. The effort required collective action, and collective action requires collective will, and collective will requires that enough people believe it matters.

      This sphere believes it matters.

      We are still waiting for the meaning to arrive.

      The Empyrean

      The web that does not exist yet.

      This is the only moment in the project that is not about the past. Everything before this was a record — of what was done to you, of what was possible, of what was aspired to. This is the record of what has not been written yet.

      There is no sphere here. No orbit, no organizing principle, no era it belongs to.

      The most honest thing that can be offered at the edge of an unrealized ideal is not a vision of what should come next — that would be pretending to know. Not a gesture toward inevitability — that would be optimism dressed as prophecy. Only this: the space before a sentence begins. The moment that belongs to whoever types next.

      Dante’s poem ends mid-sentence. His language failed. Not because the vision failed — because the vision exceeded the capacity of language to hold it.

      This project ends the same way.
